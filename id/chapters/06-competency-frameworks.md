# Bab 6: Peran, Tanggung Jawab, dan Kerangka Kompetensi

## Tujuan Pembelajaran

- Mendefinisikan peran ITSM dan tanggung jawabnya
- Mengembangkan matriks RACI yang efektif
- Merancang model kompetensi komprehensif
- Membuat jalur karier yang jelas

---

## Definisi Peran ITSM

### Peran Utama dalam Organisasi TI

| Peran | Tanggung Jawab Utama |
|-------|---------------------|
| **Service Desk Analyst** | Dukungan lini pertama, pencatatan tiket |
| **Incident Manager** | Koordinasi resolusi insiden |
| **Problem Manager** | Analisis akar masalah |
| **Change Manager** | Pengelolaan perubahan |
| **Service Level Manager** | Pemantauan SLA |
| **IT Project Manager** | Pengelolaan proyek TI |

---

## Matriks RACI

### Contoh RACI untuk Proses Incident Management

| Aktivitas | Service Desk | Incident Mgr | Tech Lead | IT Manager |
|-----------|-------------|--------------|-----------|------------|
| Pencatatan tiket | R | I | I | I |
| Diagnosis awal | R | C | C | I |
| Eskalasi | R | A | I | I |
| Resolusi | C | A | R | I |
| Penutupan | R | A | I | I |

---

## Model Kompetensi

### Hierarki Kompetensi

```
┌─────────────────────────────────────────────────────────────────┐
│                    MODEL KOMPETENSI                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Level 4: KOMPETENSI KEPEMIMPINAN                              │
│            └── Strategic Thinking, People Development           │
│                                                                 │
│   Level 3: KOMPETENSI TEKNIS                                    │
│            └── Cloud, DevOps, Security, Programming             │
│                                                                 │
│   Level 2: KOMPETENSI FUNGSIONAL                                │
│            └── ITSM, Project Management, Service Delivery       │
│                                                                 │
│   Level 1: KOMPETENSI INTI                                      │
│            └── Komunikasi, Kerja Tim, Problem Solving           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

---

## Jalur Karier

### Contoh Jalur Karier Teknis

```
Junior Developer → Developer → Senior Developer → Tech Lead → Principal Engineer → Architect
```

### Contoh Jalur Karier Manajemen

```
Team Member → Team Lead → Manager → Senior Manager → Director → VP
```

---

## Ringkasan

- Definisi peran yang jelas memastikan akuntabilitas
- RACI menjelaskan keterlibatan dalam proses
- Model kompetensi mendefinisikan kapabilitas yang diperlukan
- Jalur karier memberikan arah pengembangan
